  • Can you think of an analogy to describe HTML, CSS and the DOM? How would you describe it to your non-technical friend?
    • HTML houses the structure/elements of the page and all its contents, like a skeleton. These are in the form of tags. CSS allows you to select these elements and tell the browser how they all should look. The DOM (Document Object Model) breaks the page down and arranges the elements into a tree structure to give a map/model of the page. Here you can easily see the familial relationship of all the elements.

  • What is meant by boxifying design?
    • This is the process of taking a design layout for a webpage and visually deconstructing it, dividing it up into separate boxes. In many cases there will boxes within boxes. This is essentially chopping it up into different elements.

  • What is the box model?
    • The box model indicates what values of a container you can change. Starting with the content itself, for exmple some text, you can change the size of the space the text operates in. The next level up from that is the padding, this is the area between the content and the container. Then the container itself can be altered by changing the border. Lastly, the margin is what surrounds the border of the container, and how close it gets to outside objects.
